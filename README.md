# ГеоШтурман

Навигация для беспилотника, когда спутников нет.

Если коротко: самолёт летит, GNSS пропал (помехи, высокие широты, ущелье — неважно), а знать, где ты, всё равно надо. У нас есть радиовысотомер, барометр и заранее загруженная карта высот. Этого достаточно. Система смотрит, какой рельеф проплывает под брюхом, сравнивает его с картой и говорит: ты вот здесь, летишь туда-то, с такой-то скоростью.

Это рабочий прототип с дашбордом, который мы делали под кейс «Полёт вслепую». Не слайды — оно правда считает и правда работает в реальном времени.

---

## Что внутри, по-человечески

Аппарат знает свою высоту над уровнем моря (барометр, ~1500 м) и высоту до земли (радиовысотомер). Вычитаем одно из другого — получаем высоту самого рельефа под нами в каждый момент. Набегает «профиль» местности, как кардиограмма. Дальше берём цифровую карту высот (DEM) и ищем на ней маршрут, профиль которого ложится на наш как влитой. Перебираем все направления (0–360°) и положения, считаем корреляцию, берём максимум. Там и есть самолёт.

Дальше хитрость, без которой real-time не живёт: гонять полный перебор по карте на каждый замер — дорого. Поэтому ядро **раз в несколько секунд фиксирует** старт, курс и скорость по накопленному куску трека, а **между фиксациями позицию ведём счислением** (dead reckoning) — просто откладываем пройденный путь от последней привязки. Получается как у взрослых систем: рельеф даёт точную привязку нечасто, инерциалка/счисление держит позицию плавно между ними.

## Цифры, которые не стыдно показать

Гоняли на **реальном** рельефе Хибин (Кольский, данные Copernicus GLO-30, перепад ~830 м) при скорости 40 м/с и курсе 128°:

- захват курса и скорости — точно в цель (128° / 40 м/с);
- **CEP50 ≈ 50 м, CEP95 ≈ 130–330 м** — то есть половина решений ложится в полусотню метров;
- после захвата ошибка позиции уходит почти в ноль на прямолинейном участке;
- один пересчёт ядра — порядка 9–10 секунд на обычном ноуте (между ними позицию держит счисление).

Честная оговорка: над **плоским** рельефом (типа Подмосковья, перепад 90 м) точность закономерно падает — местности мало, цепляться не за что. Система это видит сама: достоверность снижается, и она честно переключается в режим счисления (DR) и поднимает флаг. Это не баг, это физика terrain-навигации, и мы её не прячем.

## Что в дашборде

Открываешь — и это пульт оператора, а не презентация:

- **3D-карта рельефа** — настоящий рельеф из Copernicus, по нему ползёт траектория с бортом. Крутится мышкой, зум колесом.
- **Поток NMEA** — живые строки `$GPGGA` идут прямо на глазах, выбросы подсвечиваются.
- **Курс, скорость, высоты** — компас, профиль (радиовысотомер vs карта), относительная высота.
- **Фильтры и системы** — видно, как работает Хампель (сколько выбросов отбраковано), Калман, партиклфильтр.
- **Корреляционный анализ** — 3D-поверхность строится из **настоящей** матрицы коэффициентов ядра, не нарисована.
- **Метрики** — CEP50/95, отклонения вдоль/поперёк трассы, время решения, режим (TRN/DR), интегрити.
- **Ввод данных** — выезжающее меню с тремя вкладками: вставить NMEA текстом, загрузить файлом, или задать параметры руками. Плюс «Отчёт» — выгрузка CSV+JSON.
- Переключатель **БПЛА / Воздушное судно** меняет борт на карте.

Если бэкенд недоступен (например, открыл просто статику), дашборд не падает — крутит автономный demo-поток, чтобы всегда было что показать.

## Фильтры (нас про них спрашивали)

Это каскад, а не один фильтр:

1. **Хампель / медиана** на входе — выкидывает битые строки и скачки радиовысотомера (медиана + MAD). На дашборде живой счётчик выбросов.
2. **Калман 1D** — сглаживает профиль перед сопоставлением.
3. **Партиклфильтр** — держит несколько гипотез положения; рельеф многомоден (похожих долин много), и облако частиц сходится по мере данных.
4. **Счисление + барокоррекция** — связка между терреновыми привязками, компенсация дрейфа баро.
5. **Самооценка достоверности** — по информативности рельефа и корреляции; над плоским местом честно проседает.

Подробнее — в [GeoShturman/README.md](GeoShturman/README.md).

## Как запустить

**Через Docker (как на проде):**

```bash
cd GeoShturman
cp .env.example .env
docker compose up -d --build
# открыть http://localhost
```

**Локально, чтоб поковыряться:**

```bash
cd GeoShturman/backend
python -m venv .venv && .venv/Scripts/activate   # Windows; на Linux/Mac — source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --port 8000
# открыть http://localhost:8000  ·  API — /docs
```

Хочешь реальный рельеф Copernicus вместо синтетического — подними с переменной `DEM_SOURCE=real` (тянет тайл из открытых данных AWS, кэширует):

```bash
DEM_SOURCE=real uvicorn app.main:app --port 8000
```

## Куда подавать данные кейсодателя

Кейсодатель шлёт живой поток NMEA — мы это и ждём:

- `POST /api/stream/ingest` (или WebSocket `/api/stream/ingest`) — лей строки `$GPGGA`, дашборд обновляется на лету;
- дашборд слушает `WS /api/stream/live`;
- нет внешнего источника — жми «Старт», крутится симуляция по реальному рельефу;
- разовый разбор лога — `POST /api/navigation/solve` или вкладка «Решить разом».

## Честно про границы прототипа

- Прямолинейный полёт отрабатывает отлично; на резких манёврах счисление между фиксациями подплывает — тут просится полноценный партиклфильтр на каждый шаг (есть в ядре, в стрим пока не вшит).
- Полный перебор ядра тяжеловат для слабого борта — под реальную плату нужна оптимизация (FFT-корреляция / Numba) и/или урезанный режим.
- Геопривязка считается от опорной точки региона; для боевого x,y нужна нормальная проекция (rasterio/pyproj уже подключены, регион берётся из реального тайла).

Что дальше и как встраивать в борт (PX4/ArduPilot, MAVLink, onboard-компаньон) — расписано в issue-плане и в [GeoShturman/README.md](GeoShturman/README.md).

## Стек

Python + FastAPI + NumPy/SciPy на бэке, ядро навигации в `backend/app/core`, реальный DEM через rasterio/Copernicus. Фронт — чистый HTML/CSS/JS + Three.js для 3D, без сборщиков. Развёртка — Docker + nginx (с проксированием WebSocket).

Проект гражданский: доставка в труднодоступные районы, резервная навигация, безопасность полётов, постфактум-разбор маршрута.
