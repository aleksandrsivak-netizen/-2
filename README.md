# ГеоШтурман

ГеоШтурман - проект автономной навигации БПЛА по рельефу местности. Система
использует радиовысотомер, барометрическую высоту и цифровую модель рельефа
DEM/GeoTIFF, чтобы оценивать траекторию, курс, координаты и достоверность
решения при отсутствии или деградации GNSS.

Проект состоит из двух частей:

| Компонент | Папка | Назначение |
| --- | --- | --- |
| Алгоритмическое ядро | `tercom_uav/` | Разбор входных данных, работа с DEM, построение профиля рельефа, корреляционный поиск, GPS-fusion, confidence, CLI и тесты ядра. |
| Веб-приложение | `GeoShturman/` | FastAPI backend, bridge к ядру, REST/WebSocket API, дашборд, загрузка файлов, визуализация траектории и артефактов. |
| Независимый стенд | `independent_test/` | Проверка алгоритма внешним генератором сценариев без использования внутренних симуляторов проекта. |
| Документация | `docs/` | Технические заметки по проверочным режимам и входным данным. |

Основной пользовательский сценарий для проверки: загрузить `heights.txt`,
GeoTIFF-карту, стартовые координаты, курс, скорость и частоту радиовысотомера,
получить локальную/глобальную траекторию, JSON-отчет и изображение маршрута на
рельефе.

## Возможности

- Работа с текстовым файлом высот: одна высота в метрах на строку.
- Обработка шумных, пропущенных и некорректных значений во входном профиле.
- Загрузка DEM-карты в формате GeoTIFF.
- Использование стартовой точки `(x, y)` в локальных координатах карты или в
  координатной системе GeoTIFF, если ее можно определить автоматически.
- Задание курса в градусах относительно севера: `0` - север, `90` - восток.
- Расчет шага между измерениями:

```text
sample_step_m = initial_speed_mps / radio_altimeter_hz
```

- Корреляционный поиск профиля рельефа по DEM.
- Coarse-to-fine уточнение кандидатов в режиме проверки.
- Расчет локальной траектории `x, y`.
- Расчет глобальной траектории `latitude, longitude`, если GeoTIFF содержит CRS.
- Визуализация траектории поверх карты рельефа.
- Сравнение с эталонной траекторией, если она передана.
- Вывод confidence, correlation, RMSE и диагностических предупреждений.
- Режимы GPS-assisted, GPS-degraded, GPS-rejected и TERCOM-only.

## Работа через веб-интерфейс

На главной странице расположен дашборд с 3D-рельефом, курсом, телеметрией,
confidence, профилем высот и корреляционной поверхностью.

Для проверочного набора используется вкладка **ГеоШтурман** в блоке
**Ввод данных заказчика**.

Поля вкладки:

| Поле | Описание |
| --- | --- |
| `heights.txt` | Текстовый файл высот. Одна высота в метрах на строку. |
| `GeoTIFF` | DEM-карта рельефа. Желательно с CRS для получения глобальных координат. |
| `Старт X, м` | Начальная координата X. |
| `Старт Y, м` | Начальная координата Y. |
| `Курс от севера, градусы` | Направление движения. |
| `Скорость, м/с` | Начальная скорость аппарата. |
| `Частота, Гц` | Частота радиовысотомера. |
| `Радиус поиска, м` | Ограничение области поиска вокруг стартовой оценки. |
| `Эталонная траектория` | Необязательный CSV/GeoJSON для расчета ошибок. |

После нажатия **Запустить проверку** backend возвращает:

- локальный CSV;
- глобальный CSV;
- визуализацию;
- JSON-отчет;
- диагностические предупреждения по входным данным;
- confidence/correlation/RMSE.

Ссылки на артефакты открываются через `/api/artifact-view/...`, поэтому CSV,
JSON и изображение можно смотреть прямо в браузере.

## Формат входных данных

### heights.txt

Файл высот содержит одну высоту на строку:

```text
441.77
438.30
440.70
439.89
...
```

Допускаются пустые строки, `nan`, `inf`, текстовые ошибки и пропуски. Такие
элементы не используются в корреляции и фиксируются в `warnings`.

### GeoTIFF

GeoTIFF используется как DEM-карта. Поддерживаются:

- локальные метрические карты;
- карты с affine transform;
- карты с CRS;
- карты, где координаты можно перевести в `lat/lon` через `rasterio` и `pyproj`.

Если CRS отсутствует, система строит локальную траекторию, а глобальные
координаты помечаются как недоступные.

### Стартовая точка

Старт `(x, y)` используется как исходная оценка положения. В режиме `auto`
ядро пытается определить, являются ли координаты:

- локальными координатами карты;
- координатами пикселя;
- координатами в CRS GeoTIFF.

### Курс

Курс задается в градусах относительно севера:

```text
0   = север
90  = восток
180 = юг
270 = запад
```

В локальной метрической системе:

```text
x = x0 + sin(course_deg) * distance_m
y = y0 + cos(course_deg) * distance_m
```

### Скорость и частота

Если расстояние между соседними измерениями не задано напрямую, оно считается:

```text
sample_step_m = initial_speed_mps / radio_altimeter_hz
```

Например, при скорости `100 м/с` и частоте `5 Гц` шаг профиля равен `20 м`.

### Эталонная траектория

Эталонная траектория необязательна. Если она передана, система считает:

- среднюю горизонтальную ошибку;
- максимальную горизонтальную ошибку;
- RMSE;
- смещение старта;
- смещение конца.

## Как работает алгоритм

1. **Чтение входа.** Высоты читаются из текста, приводятся к `float`,
   нечисловые значения помечаются как проблемы входных данных.
2. **Построение дистанционной шкалы.** Для каждой высоты считается дистанция
   от старта по шагу `sample_step_m`.
3. **Построение начальной траектории.** По стартовой точке и курсу формируется
   линия движения в локальных координатах.
4. **Сэмплирование DEM.** Для каждой точки траектории извлекается высота карты.
5. **Сравнение профилей.** Входной профиль высот сравнивается с DEM-профилем
   по correlation/RMSE/MAE и нормализованной форме.
6. **Поиск лучшего совпадения.** В окне поиска перебираются смещения и, при
   необходимости, варианты курса.
7. **Coarse-to-fine.** Сначала проверяется грубая сетка кандидатов, затем
   лучшие кандидаты уточняются на мелком шаге.
8. **Оценка качества.** Confidence строится по корреляции, RMSE, форме профиля
   и неоднозначности найденного максимума.
9. **Формирование результата.** Генерируются CSV, GeoJSON, PNG и JSON-отчет.

## GPS/GNSS-сценарии

Система поддерживает три основных режима.

### GPS включен и работает нормально

GPS используется как внешняя оценка положения. Рельефная привязка остается
независимой проверкой. Если GPS и ГеоШтурман согласованы, координаты
объединяются через weighted fusion, а окно поиска можно сузить.

Статус:

```text
GPS_HEALTHY_ASSISTED
```

### GPS включен, но дает некорректные данные

Если GPS старый, скачет, имеет плохой HDOP, мало спутников или противоречит
уверенной рельефной привязке, он отклоняется. ГеоШтурман переходит в режим
повторного захвата по DEM и снижает доверие к GPS.

Статусы:

```text
GPS_DEGRADED
GPS_REJECTED_REACQUIRE
DATA_STALE
DATA_INVALID
```

### GPS отключен

Работает прежний режим рельефной навигации без GNSS. Положение определяется по
профилю высот и DEM, между фиксациями используется счисление пути.

Статус:

```text
GPS_OFF_TERCOM_ONLY
```

## Неточные, запоздавшие и поврежденные данные

Система не принимает входные данные без проверки.

| Проблема | Поведение |
| --- | --- |
| Нечисловые высоты | Значения исключаются из сопоставления, предупреждение попадает в `warnings`. |
| `nan`/`inf` | Исключаются из расчета, валидные точки продолжают использоваться. |
| Шум радиовысотомера | Профиль сглаживается, качество отражается в RMSE и confidence. |
| Плоский рельеф | Confidence снижается, решение считается менее надежным. |
| Запоздавшие NMEA/GPS | Выставляется `DATA_STALE`, источник получает меньший вес или отклоняется. |
| Данные не по порядку | Выставляется `is_out_of_order`, подозрительный пакет не становится основной оценкой. |
| Скачок GPS | GPS отклоняется, включается reacquire по рельефу. |
| Выход за DEM | Возвращается ошибка входных данных, траектория не считается достоверной. |

## Частые проблемы

### `python` открывает Microsoft Store

Используйте launcher:

```powershell
py --version
py -m pip --version
```

### Не открываются CSV/JSON артефакты

Используйте ссылки вида:

```text
/api/artifact-view/<run_id>/trajectory_local_csv
/api/artifact-view/<run_id>/result_json
```

Прямые `/api/artifacts/.../*.csv` предназначены для программной загрузки.

### Нет глобальных координат

Проверьте, что GeoTIFF содержит CRS. Без CRS система корректно строит локальные
координаты, но не может надежно получить `latitude/longitude`.

### Низкий confidence

Типовые причины:

- плоский или повторяющийся рельеф;
- мало точек профиля;
- высокая шумность высот;
- стартовая точка далеко от фактического участка;
- слишком маленький радиус поиска;
- неверно задан курс или шаг измерений.

### Сервер уже занят на порту 8000

Проверьте процесс:

```powershell
Get-NetTCPConnection -LocalPort 8000 -State Listen
```

Запустить на другом порту:

```powershell
py -m uvicorn app.main:app --host 127.0.0.1 --port 8001
```

## Ограничения

- Рельефная навигация плохо различает плоские и повторяющиеся участки.
- Качество зависит от точности DEM и синхронизации высот.
- Для больших карт полный поиск может быть дорогим по времени, поэтому
  используются окно поиска, ограничение гипотез и coarse-to-fine.
- При отсутствии CRS глобальные координаты не вычисляются.
- GPS используется только после проверки качества, возраста и согласованности
  с рельефной привязкой.

## Гражданское назначение

Проект предназначен для гражданских сценариев:

- резервная навигация БПЛА при потере GNSS;
- доставка грузов и медикаментов в труднодоступные районы;
- мониторинг инфраструктуры;
- геологоразведка и картографирование;
- анализ траекторий после полета.
